# 메시지 수신 플로우 슬라이드 지시서 (1장)

> 클라이언트에서 CS_CHAT이 도착했을 때, 서버 내부에서 일어나는 전체 흐름을 1장으로 표현

---

## 슬라이드 구성

- **제목**: `CS_CHAT 수신 → SC_CHAT 브로드캐스트 플로우`
- **레이아웃**: 수평 Swim Lane (레이어별 띠로 구분)
- **배경**: 짙은 남색 또는 검정
- **화살표**: 흐름 방향에 따라 색상 구분
  - 수신 흐름(CS) : 파란색 화살표
  - 내부 처리 : 흰색/회색 화살표
  - 송신 흐름(SC) : 초록색 화살표
  - 실패/종료 : 빨간색 화살표

---

## 콜아웃 박스 (슬라이드 여백에 배치)

슬라이드 오른쪽 또는 하단에 아래 3개 설명 박스를 작게 배치:

```
┌─────────────────────────────────┐
│ 🔐 복호화 타이밍                │
│ PlayerMap에서 FK를 꺼낸 직후    │
│ DeCoding(FK) 수행               │
│ → Worker Thread는 암호화 상태   │
│   그대로 ContentsQ에 넣음       │
└─────────────────────────────────┘

┌─────────────────────────────────┐
│ ⚡ Lock은 단 1번                │
│ onRecv()의 ContentsQ push 시    │
│ mQLock 하나만 사용              │
│ → 게임 로직 구간에 락 없음      │
└─────────────────────────────────┘

┌─────────────────────────────────┐
│ 📡 Send 팬아웃                  │
│ CS_CHAT 1개 수신                │
│ → 주변 N명에게 SC_CHAT 전송     │
│ → Send TPS ≈ Recv TPS × 3.1    │
└─────────────────────────────────┘
```

## 주의사항

- Swim Lane 경계선을 얇은 회색 선으로 구분
- 실패 분기(빨간 화살표)는 오른쪽으로 빠지게 표현
- `DeCoding(FK)` 와 `EnCoding(FK)` 텍스트는 눈에 띄는 색(노란색 등)으로 강조
- 전체 흐름이 왼쪽→오른쪽 또는 위→아래 한 방향으로 읽히도록 배치
